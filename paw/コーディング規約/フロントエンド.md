# フロントエンドコーディング規約

## 技術スタック

- Next.js 16（App Router）/ React 19
- TypeScript
- Tailwind CSS v4
- shadcn/ui
- Auth0（`@auth0/nextjs-auth0`）
- pnpm

## ディレクトリ構成

```
src
├─ app
│  ├─ (authorized)          # 認証必須ページ
│  │  └─ <ページ名>
│  │     ├─ _components     # そのページ専用のコンポーネント
│  │     │  └─ index.tsx
│  │     └─ page.tsx
│  └─ (unauthorized)        # 未認証ページ
├─ components
│  ├─ common                # 汎用コンポーネント
│  ├─ layout                # レイアウトコンポーネント
│  └─ ui                    # shadcn/ui 専用（自動生成のみ、手編集しない）
├─ features                 # 機能単位のモジュール
│  └─ <機能名>
│     ├─ components         # 機能固有コンポーネント
│     │  └─ <ComponentName>
│     │     └─ index.tsx
│     ├─ actions.ts         # Server Actions
│     ├─ api.ts             # データフェッチ関数
│     ├─ const.ts           # 定数
│     └─ types.ts           # 型定義
├─ lib                      # 外部ライブラリの設定
├─ util                     # 汎用ユーティリティ
│  └─ fetcher               # APIクライアント
└─ proxy.ts
```

## 命名規則

| 種別                   | 命名例                                                 |
| ---------------------- | ------------------------------------------------------ |
| コンポーネントファイル | `TransactionTable/index.tsx`                           |
| コンポーネント（関数） | `TransactionTable`（パスカルケース）                   |
| Server Actions         | `createTransactionAction`（`Action` サフィックス）     |
| ActionState 型         | `CreateTransactionActionState`                         |
| APIレスポンス型        | `TransactionDetailResponse`（`Response` サフィックス） |
| 定数ファイル           | `const.ts`                                             |
| 型ファイル             | `types.ts`                                             |

## features の設計

機能ごとに `features/<機能名>/` ディレクトリを作成する。

### api.ts

APIへのデータフェッチ関数のみを置く。`fetcher` ユーティリティを使う。

```ts
import { fetcher } from "@/util/fetcher";

export async function getTransactions() {
  return await fetcher.get("/transactions");
}
```

### actions.ts

Server Actions のみを置く。ファイル先頭に `"use server"` を宣言する。

```ts
"use server";

import { fetcher } from "@/util/fetcher";
import { redirect } from "next/navigation";

export type CreateTransactionActionState = {
  success?: boolean;
  errors?: {
    fieldName?: string;
    global?: string;
  };
};

export async function createTransactionAction(
  _prev: CreateTransactionActionState,
  formData: FormData,
): Promise<CreateTransactionActionState> {
  // バリデーション → fetcher 呼び出し → redirect or return state
}
```

- Server Actions はバリデーション → APIコール → リダイレクト（または状態返却）の流れで実装する
- バリデーションエラーは `errors` オブジェクトにフィールド単位で格納する
- グローバルエラーは `errors.global` に格納する

### types.ts

APIレスポンスの型定義のみを置く。APIレスポンスの型名には `Response` サフィックスをつける。

```ts
export type TransactionDetailResponse = {
  id: number;
  transactionName: string | null;
  amount: number;
};
```

## fetcher ユーティリティ

`@/util/fetcher` の `fetcher` オブジェクトを通じてAPIを呼び出す。生の `fetch` は使わない。

```ts
fetcher.get("/path");
fetcher.post("/path", body);
fetcher.put("/path", body);
fetcher.delete("/path");
```

- タイムアウト・エラーハンドリングは fetcher 内部で処理される
- レスポンスは `{ body, statusCode }` または `{ error, statusCode }` の形で返る
- エラー判定は `"error" in result` で行う

## コンポーネント規約

- コンポーネントは名前付きエクスポートを使う（`export const Foo = ...`）
- Props の型は `type Props = { ... }` でコンポーネントと同じファイルに定義する
- ページ（`page.tsx`）のデフォルトエクスポートは例外として許容する

```tsx
type Props = {
  account: AccountSummaryResponse;
};

export const AccountCard = ({ account }: Props) => {
  return <Card>...</Card>;
};
```

## shadcn/ui

- `components/ui/` ディレクトリは shadcn/ui の自動生成専用とする
- `components/ui/` のファイルを直接編集しない
- shadcn コンポーネントは `@/components/ui/<name>` からインポートする

## スタイリング

- スタイルは Tailwind CSS クラスで記述する
- 条件付きクラスの結合には `cn()` ユーティリティ（`@/lib/utils`）を使う
- インラインスタイル（`style={}` 属性）は使わない

```tsx
import { cn } from "@/lib/utils";

<div className={cn("base-class", condition && "conditional-class")} />;
```

## 認証

- Auth0 の設定は `@/lib/auth0` に集約する
- `fetcher` 内部でアクセストークンを自動付与するため、APIコール時にトークンを手動で渡す必要はない
- カスタムアクセストークンが必要な場合のみ `fetcher.get(path, {}, accessToken)` で渡す

## 型安全性

- `any` は使わない
- 型アサーション（`as`）は最小限にとどめる
- APIレスポンスの型は `types.ts` で明示的に定義する
- 定数オブジェクトから型を生成する場合は `keyof typeof` を活用する

## コーディングスタイル

- インデント：2スペース
- 末尾カンマあり
- セミコロンあり
- フォーマットは Prettier に従う（`prettier-plugin-tailwindcss` によりクラスの順序も自動整形）
